프로젝트 보안 강화 결과 및 안내 사항
프로젝트의 API 키와 민감한 정보를 안전하게 보호하기 위한 모든 작업을 완료했습니다.

변경 사항 요약
환경 변수 분리: 
server.js
에서 사용하던 API 키를 
.env
 파일로 분리했습니다.
Git 추적 제외: 
.env
 파일과 Firebase 설정 파일을 
.gitignore
에 등록하여 향후 유출을 방지했습니다.
설정 템플릿 제공: 협업자를 위해 
.env.example
 및 
firebase-config.sample.js
 파일을 생성했습니다.
의존성 추가: 환경 변수 로드를 위해 dotenv 패키지를 추가했습니다.
커밋 완료: Parking_Web 메시지로 변경 사항을 저장했습니다.
중요: 이미 유출된 기록 삭제하기
컴퓨터에 변경 사항을 적용하고 커밋했지만, 과거 커밋 기록에는 여전히 API 키가 남아 있습니다. 이를 완전히 삭제하려면 다음 단계를 따라주세요.

1단계: API 키 재발급 (가장 중요)
현재 유출된 키는 무효화해야 합니다. 각각의 관리자 페이지에서 키를 새로 발급받으세요.

공공데이터포털
카카오 개발자 콘솔
Firebase 콘솔
2단계: Git 기록 삭제
git-filter-repo 또는 명령어를 사용하여 특정 텍스트가 포함된 모든 기록을 지울 수 있습니다. (추가 안내가 필요하시면 말씀해주세요!)

프로젝트 실행 방법 (GitHub에서 다운로드 후)
나중에 다른 곳에서 프로젝트를 클론(Clone) 받거나 다운로드했을 때 실행하는 순서입니다.

1. 패키지 설치
터미널에서 프로젝트 폴더로 이동한 후 다음 명령어를 실행하여 필요한 도구들을 설치합니다.

bash
npm install
2. 환경 변수 설정 (.env)
프로젝트 루트 폴더에 
.env
 파일을 새로 만듭니다.
.env.example
 파일의 내용을 복사해서 붙여넣습니다.
GONGU_API_KEY 부분에 재발급받은 실제 API 키를 입력합니다.
3. Firebase 설정 설정
scripts/firebase-config.sample.js
 파일을 복사하여 
scripts/firebase-config.js
를 만듭니다.
파일 안의 firebaseConfig 객체에 본인의 Firebase 프로젝트 정보를 입력합니다.
4. 카카오 맵 도메인 제한
[카카오 개발자 콘솔] -> [내 애플리케이션] -> [플랫폼] -> [Web] 탭에서 현재 실행 중인 사이트 주소(예: http://localhost:5500)를 등록해야 지도가 정상적으로 나옵니다.
5. 서버 실행
bash
node server.js
서버가 실행되면 웹 페이지에서 주차장 데이터를 정상적으로 가져올 수 있습니다.
